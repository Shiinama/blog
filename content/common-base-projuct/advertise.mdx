# 开发者如何获得长期职业与财务安全感？

在国内，可能99%的开发同学都是担任螺丝钉的角色，仅仅只是或小或大的问题。但很显然螺丝钉是有一个使用期限的，当生锈了就会被报废，这并不是在贩卖焦虑，而是在描述一个市场规律，有哪个打工人敢100%的保证自己未来一定不被裁?

解法很多：

1. 随便找个厂慢慢往上爬、多积累一些人脉、多存点钱、成为M序列。
2. 做个人IP，影响力赚钱。
3. 做副业，尝试把副业变成主业。

第一点是成为了M序列了之后，跟公司有深度绑定关系了，本身培养合适的人是很难的，培养M序列的人更难，开除你的代价太高了。
第二点和第三点是因为你自己给自己发工资，你焦虑的点从害怕打工被裁变为了怎么赚到比打工更多的钱。

这只是随便列了三种，世界上通往成功的道路不止一条。**解法的核心在于：满足自己对于钱的安全感，并且这种安全感的满足是需要长期稳定的。**

## 那最适合开发的路是什么那？

其实现在有非常多的付费副业群，教程序员各种副业，比如：公众号爆文、小红书相册、摆摊......

毋庸置疑，这些教程都很有道理，也条理清晰，也很吸引人，可能最后也真的能做成什么，大家的说辞也出奇的一致，就是什么只要努力，有行动力吧啦吧啦的。

可是我们是工程师啊，在我的理解中，我们应该扮演更重要的角色。让我们回溯到人类社会的起源，从原始社会讲起：

在远古时代，人类面临着生存的巨大挑战。那时，最聪明的部落成员不会满足于仅仅采集更多的浆果或猎杀更多的野兽。相反，他们开始思考如何提高效率：如何制作更好的工具来采集食物，如何设计更有效的陷阱来捕获猎物。这些人就是当时的"工程师"，他们创造的工具极大地提高了整个部落的生产力。

随着时间的推移，这种模式一直在延续。农业革命中，发明犁的人比辛勤耕种的农民对社会的贡献更大。工业革命时期，设计蒸汽机的工程师比操作机器的工人更能推动社会进步。

**我想表达的意思就是：作为开发，本身我们就是距离工具原理更近的那一侧，我们本身学到的所有技能在最开始的初衷就是为了创造工具，而不是被现在傻逼的国内环境扭曲成这个样子，然后你现在要告诉我，你要放弃自己本身的优势去做一些所谓看起来很赚钱的副业？**

所以我建了个付费社群，提供一条路：通过技术的方式来做项目，加大技术小伙伴做项目的成功率，实现长期现金流项目，稳定收入结构。

我来做这个社群的优势是在于：

1. 我就在AI赛道创业，也正在做现金流的项目，而且我也成功了，所有讲的东西都是在市场经过实践和验证的。
2. 世界上成本最低的开发部署整套方案。
3. 世界上最好的AI项目模版和开发思路。
4. 以开发的视角讲怎么获取项目流量，以及怎么寻找流量杠杆。
5. 开发、流量、产品、付费、商业模式，我能连在一起讲。

> 我其实写的课程跟我创业要做的事情是连在一起的，要不就是未来一定要做的事情，要不就是已经做过的事情~

最终能达到的结果：最短30分钟能启动一个项目，每天最多1小时运营，通过不断的短频快的尝试，加大成功率，以大量项目尸体堆起来，找到那几个真正的现金流项目。
