# App开发大纲

## 第一部分：基础篇

### 1. Expo 生态与核心概念

- **什么是 Expo？** 为什么选择它来开发 React Native 应用。
- **Expo 生态概览**: Expo SDK, Expo Go, EAS (Expo Application Services)。
- **核心优势与权衡**: 快速启动、跨平台一致性 vs. 原生限制。

### 2. 工作流选择与环境配置

- **Managed vs. Bare 工作流**: 如何根据项目需求做出正确决策。
- **平滑过渡**: 何时以及如何从 Managed “eject” 到 Bare 工作流。
- **开发环境搭建**: 安装 Expo CLI，配置模拟器和真机调试。
- **项目初始化与配置文件**: `create-expo-app` 与 `app.json` 详解。

## 第二部分：实战篇

### 3. 核心功能实现：推送通知

- **原理**: APNs (Apple) 与 FCM (Google) 工作机制。
- **集成 Expo Push Service**: 从设置到发送第一条通知。
- **处理通知**: 在前台、后台、应用关闭时响应通知。

### 4. 高级功能实现：应用内购买 (IAP)

- **平台差异**: App Store Connect 与 Google Play Console 的设置。
- **实现方案**: 使用 `expo-in-app-purchases` 或 `react-native-iap`。
- **核心流程**: 实现商品查询、购买、消耗和收据验证。

### 5. 终极扩展：原生模块开发 (Bare Workflow)

- **何时需要**: 当 Expo SDK 无法满足需求时。
- **iOS 原生模块**: 使用 Swift 或 Objective-C 编写原生功能并桥接到 JS。
- **Android 原生模块**: 使用 Kotlin 或 Java 编写原生功能。
- **集成第三方原生库**: 如何链接没有 Expo 支持的库。

## 第三部分：部署篇

### 6. 构建与发布 (EAS Build)

- **云端构建**: 使用 EAS Build 构建 iOS 和 Android 应用包。
- **`eas.json` 配置**: 为开发、预览、生产环境创建不同构建档案。
- **证书管理**: EAS 如何简化复杂的签名和证书流程。
- **应用商店提交**: 自动化上传到 TestFlight 和 Google Play。

### 7. 热更新与维护 (EAS Update)

- **OTA 更新**: 无需审核，即时向用户推送代码更新。
- **更新渠道 (Channels)**: 管理 `staging` 和 `production` 等不同环境的更新。
- **发布与回滚**: 安全地发布新版本并处理异常情况。
